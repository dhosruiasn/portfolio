## 目標

- 開燈後接近參考圖二：四周維持深棕黑，亮度集中在燈泡下方與字面中央。
- 避免原本整面偏亮、偏平的橘紅漸層；保留暗部，讓燈與白熱字成為視覺焦點。
- 顆粒必須在一般桌面螢幕可直接看見，質感接近高 ISO 底片／印刷網點，而不是很淡的數位 noise。

### 後續協作原則

- 當需求涉及寫實材質、空間延伸或 CSS 無法無縫重建的背景時，應主動與 Image／imagegen 協作，不等待使用者再次提醒；生成素材、提示與使用方式都要記錄在本檔。
- 背景之後預計加入光線流動，因此 Hero 背景、向下延伸地面、文字、燈具、卡片、噪點與動態光場必須保持獨立圖層。後續可在兩段背景上使用連續的光流遮罩或亮度動畫，不把流動效果烘焙進靜態圖。

## 展廳隱藏式崁燈光場（已實作）

- 光流不是烘焙進背景，也不位移或縮放背景圖；Hero 與作品區保留兩個 SVG 顯示面，但由同一次進場順序交接，避免點陣圖在動畫時失焦、產生壓縮接縫。
- 視覺語意是「光源藏在弧形牆縫／燈槽後面，牆面受到間接照明」，不是可見的細線光束或霓虹燈條。
- Hero 使用一條連續路徑：暖光從左端進入，沿下凹弧線由左往右，再貼右側邊界向下。取消先前左右對稱的兩條光路。
- SVG clip-path 以背景原圖像素取樣後擬合：左右端 `y=31–33/1000`、中心 `y=220/1000`，使用拋物線 `M 0 31 Q 500 409 1000 33`。遮罩只保留弧線下方的反射光，藍線以上天花維持暗部，不可再靠目測估線。
- 暖光改為兩層橘黃光場：外暈 `#ff6a00`、寬核心 `#ffb21c`；作品區使用同系 `#ff6900`／`#ffb11a`。不可使用奶油白或偏粉白的燈色。
- 光場採 24 秒水流式 easing。Hero 光只執行一次並停在右側交接端；作品區進入視窗時才從同一側接續，不使用固定延遲或各自循環。
- 四張作品不是整框呼吸。每張只保留約 24% 周長的局部暖白 glint，以 conic-gradient 沿圓角邊界掠過；順序由右側下降光先接第四張，再依序往第三、第二、第一張傳遞。卡片 glint 使用 18 秒循環與 3 秒相位差，非接棒時仍保留 0.12–0.15 的低亮流動，避免使用者剛好錯過短暫峰值。
- 光場使用 screen 混合；卡片維持在光場上層，保留展品陳列感。
- 光流只在開燈時間軸後淡入；關燈前樣式不變。`prefers-reduced-motion` 下停止位移，只保留極淡靜態反光。
- 作品區改用 v5 無損寬圖，取消原本的 `brightness`／`saturate` 濾鏡，避免把 JPEG 壓縮與顆粒一併放大。

## 導覽列毛玻璃

- Nav 改用與 Hero 相同語彙的半透明毛玻璃：`blur(18px) saturate(0.92)`、薄白上下反光、極淡 16 欄柱狀折射與 soft-light fractal noise。
- 噪點 opacity 為 0.1，只增加材質，不影響文字清晰度；手機展開選單沿用同一玻璃底色與 blur。

- 背景基底：`#190503`–`#4b1006` 的深棕紅。
- 主光池：中心偏上、窄於舊版，橘黃核心向深紅快速衰減。
- 暗角：橢圓 vignette，邊緣最高約 72% 黑。

## 標題入場動畫

- 原始標題是 PNG 點陣圖，不具可沿路徑描邊的 SVG path，因此不採用假的逐筆 stroke 動畫。
- 圖三的水平切線來自上下兩個 clip-path 分層；已完全移除分行裁切。
- 改用整張字圖的單一 102° 斜向柔邊光洗 mask，顯影時上下兩行共用同一個連續 alpha 場，不可能再產生中間接縫。
- 進一步移除整張模糊 glow clone 與 mask 容器 brightness filter，避免大型 PNG 在 GPU 分塊合成時產生 tile seam；光暈只保留字圖本身的多層 drop-shadow。
- 顯影完成後才淡入既有 sheen；拉繩重播會重設單一 mask-size。

## 燈具進場

- 燈罩、燈泡與珠鍊作為同一 fixture 從畫面上方 `-56vh` 落下。
- 第一次超過定位點 18px，接著以 `7° → -3.5° → 1.5° → 0°` 做阻尼擺動。
- 擺動尚未完全停止時拉動珠鍊並開燈，動作彼此重疊，避免像三段獨立動畫。

## 游標碰撞擺動

- 僅在 `(hover: hover) and (pointer: fine)` 且未啟用 reduced-motion 時生效。
- 碰撞區依透明 PNG 內真正的燈罩範圍設定，不使用整個方形圖片盒。
- 游標方向、速度與碰撞側轉成角速度；最大角度 ±12°，彈簧回正並以阻尼逐漸停止。
- 燈罩與燈泡共用 fixture 旋轉；光束角度同步為燈具角度的 72%。順時針傾斜時，開口法線與地面光池往左，因此光池位移使用相反符號，符合真實投射方向。
- 珠鍊是 fixture 內的第二個鐘擺：燈罩加速時會因慣性延遲並反向擺動，游標直接碰珠鍊也能獨立施加衝量；鏈子動量只會少量回傳燈罩。
- 關燈珠鍊 PNG 頂端原本殘留約 8% 的燈罩底緣像素；該區已用 clip-path 隱藏，避免獨立擺動時帶走一塊燈罩。曾加入的 CSS 圓角矩形 anchor 會形成多餘形狀，現已完全移除。
- 該 8% 同時也是拆鏈後燈罩缺口的原始補片，不能直接丟棄；現以同一張素材的頂端區塊固定疊回 fixture 前緣（z4），珠鍊本體位於燈罩本體與前緣反光之間（z3）。分界不可回到 10%，否則會把第一顆珠子的上半部留在固定補片裡，造成珠鏈像被壓在燈罩後方。
- 關燈與開燈燈罩／珠鍊的造型及裁切起點不同，不能永久疊在一起。開燈瞬間會讓整個關燈燈罩、關燈珠鍊與缺口補片同步淡出，亮燈燈罩與亮燈珠鍊同步淡入；任一穩定狀態都只保留一組相符的燈罩、珠鍊與接點。若保留關燈燈罩在亮燈圖下方，透明區會露出關燈狀態較厚的底緣，並在拆鏈區形成假的矩形缺口。
- 落下／開燈時間軸完成前暫停游標物理，避免兩套 transform 互相覆寫。

## Hero 轉場收邊

- 不使用格線漸隱或底部霧化；格線完整延伸到 Hero 的玻璃底邊。
- 不使用額外的 13／18px 矩形截面，也不製造高低差；16 片 column 共用同一水平下緣。
- 立體感完全由玻璃本體內的反光建立：左右柱面明暗、底部 1px 鏡面反射，以及靠近下緣的柔和內反射／暗折射。
- 下邊緣保持乾淨、薄且連續，畫面中不會出現像後加上去的封口零件。

## 驗收重點

1. 1440×900 與 1920×1080：左右角落應明顯比中央暗，不能整面同亮度橘色。
2. 不放大截圖也能看到細顆粒，但顆粒不能搶過標題筆畫。
3. `DIGITAL WORKS` 比修改前明顯放大，仍完整置中且不產生水平捲軸。
4. 拉繩可重播；開燈前不應露出光場或標題。

## 弧形控制板與穩定性（2026-07-12）

- 開發環境使用 `http://127.0.0.1:5173/portfolio/?digitalEdit=1#digital-works` 開啟控制板；正式 build 不會帶入控制板。
- 畫面會顯示藍色虛線弧形與左／中／右三個定位點。座標以背景原圖 `1672×941` 為準，不跟著瀏覽器 CSS 像素改變。
- `左端Y`、`中央Y`、`右端Y` 控制三個對齊點；`控制X` 調整弧形最低點的水平位置；`光寬`、`光高`、`秒數`、`流光` 分別控制暖光團尺寸、單次行進時間與透明度。
- 控制板只改目前頁面的記憶體狀態，重新整理即還原；對齊後按「複製弧線參數」，把 JSON 貼回協作對話再寫入預設值。先前卡片控制板改由 `?cardEdit=1` 單獨開啟，避免遮擋弧形對齊。

先前偶發閃爍並非單一圖片壞掉，而是三個合成問題疊加：大型 SVG 路徑同時套用 `blur()`／`drop-shadow()` 並位移時，瀏覽器會重新切割 GPU 合成圖磚；GSAP 又對全畫面光層動畫 `filter: brightness()`；ScrollTrigger 的 `onEnterBack` 還會在回捲時無條件重播開燈時間軸。這些操作會在某一幀同時重建大面積圖層，呈現為亮度跳動、水平切片或短暫閃白。

目前修正方式不改變既有構圖：

- 移除全畫面的動態 brightness filter；只讓 GSAP 控制光層容器 opacity。
- 把模糊路徑改成單一 SVG 橘黃 radial-gradient 光團；光團只用 SVG `animateMotion` 移動，不再對大面積元素做動態 blur／drop-shadow。
- `onEnterBack` 只在尚未開燈時重播，避免已亮燈畫面先被重設成 opacity 0 再亮起。
- 同一元素的 transform 只交給一套動畫控制：SVG 光團由 SMIL 管位移，父層由 GSAP 管顯示；不再用 CSS `will-change: transform` 強制建立另一個互相競爭的合成層。
- `prefers-reduced-motion` 下停止光團位移並保留低亮靜態反光。

### Hero／作品區光源交接修正

- 原本兩個 `animateMotion` 都有自己的無限循環與啟動時間。捲動速度不會跟動畫週期相同，因此作品區光團可能先跑到左側，或在使用者觀看時突然從右側重新開始，形成沒有來源的圓形亮斑。
- Hero 與作品區現在都改成 `begin="indefinite"`、單次播放及 `fill="freeze"`。Hero 開燈時啟動一次；作品區只有在 section 到達 viewport 92% 時才啟動。
- 交接時以 0.8 秒 crossfade 將 Hero 光淡出、作品區光淡入。Hero 光停在右下交接端，因此 180vh sticky 滑動區不會把光整段吃掉。
- Hero 右牆轉角改成向畫面外側繞行的 cubic curve，保持橢圓光團 `rotate="auto"`，形成有半徑的 90 度轉彎；作品區從右側垂直下降後用 quadratic curve 轉向卡片上緣。
- 作品光完成後停在畫面左側外，不回到右邊重新生成；持續的展場導覽感由四張卡片既有的局部邊緣 glint 接手，不再保留一顆無限繞行、難以辨識路徑的背景光圈。
- 歷史版本曾把作品區光團由 `182×128` 改成 `240×70` 長橢圓；此 rigid ellipse 方案已被後述的三層 path stroke 完全取代。

### 光源路徑控制板

- `?digitalEdit=1` 除了原本的弧線、光團尺寸與速度，新增 Hero 右牆的 `外繞X／控點Y／轉直Y／牆底Y`，以及作品區的 `下降Y／轉彎Y／轉彎X／出口Y`。

- 「複製光源路徑參數」會一併輸出兩段路徑座標，調整仍只存在目前頁面，重新整理即回到程式預設值。
- 作品區不再把 SVG 路徑拉滿整個 `170vh` section；預設使用 `100vh` 路徑面，並提供 `路徑高`（60–150vh）控制。這可避免 Y 座標被 section 高度放大，導致節點看似無法移到背景弧線。
- 作品區水平段改成精確通過 `右端Y／中央Y／左端Y` 的 quadratic curve，三點範圍開放到 `-40px`，可直接貼齊背景下弧形；`下降Y` 則可到 `-80px`，方便把右牆交接藏到畫面上緣之外。
- 2026-07-12 依控制板截圖採用作品區參數：`下降Y=-80`、`右端Y=64`、`右端X=972`、`中央Y=162`、`左端Y=54`、`路徑高=76vh`。
- 複製按鈕加入 Clipboard API 失敗時的 fallback；若 in-app browser 不允許寫入剪貼簿，JSON textarea 會自動全選並顯示「請按 ⌘C」。textarea 也可直接點擊全選。
- Hero 右下角不再以垂直線直接結束。路徑使用四分之一圓的 cubic Bézier 近似值 `0.55228475 × radius`，進彎保持垂直切線、出彎保持水平切線；光團的 `rotate="auto"` 因此會沿整段曲率連續旋轉。
- 控制板新增 `底彎R`，預設 `210px`、範圍 `60–360px`。`牆底Y` 控制出彎高度，`底彎R` 控制提早多少距離開始轉向；半徑會自動限制在右牆有效高度內，避免控制點交叉造成折線。
- 光源交棒完全取消作品區 ScrollTrigger。開燈時 Hero motion 立即開始，經過 `flowDuration - 0.8s` 後由同一時間軸自動啟動作品區 motion，並用 0.8 秒 crossfade 交接；使用者目前滑到哪個 section、往上或往下滑，都不會觸發、暫停或重播光源。
- 只有離開整個 Digital Works 區域回到上方，或重新拉燈繩時，才會清除尚未執行的交棒 timer。`flowDuration` 控制 Hero 單次行進時間，也同步決定作品區接棒時刻。
- 作品區路徑原本是「右側 quadratic 轉角＋單一 quadratic 下弧線」。雖然轉角入口／出口各自有切線，但兩段交界的切線方向不同，`rotate="auto"` 仍會在右端控制點瞬間跳角度。
- 現改成三段連續 cubic Bézier：右側使用 `0.55228475` 的橢圓四分之一轉角；右端→中央與中央→左端各用一段 cubic，handle 為水平跨度的 `36%`。右端、中央、左端均保持水平共切線，光團會像水流般連續轉向。
- 進一步檢查截圖後確認，生硬感仍來自「固定 `240×70` 橢圓沿路徑旋轉」，而不是 cubic 本身；橢圓無法隨彎道變形，`fill="freeze"` 又會在左端留下固定黃色光塊。
- 作品區現改為三層圓頭 path stroke：外層 `120px` 橘紅環境光、中層 `62px` 暖橘反射、核心 `18px` 橘黃亮帶。三層共享 `pathLength=1` 與 `stroke-dasharray="0.24 0.76"`，動畫只移動 dash offset，因此亮帶會真正沿 cubic 彎曲，不使用 rigid ellipse 或動態 blur。
- 作品光抵達左端前 0.85 秒開始淡出，完成後整個作品光層 opacity 為 0，不再因 `fill="freeze"` 在左側殘留一段亮光。
- `digitalEdit` 對齊模式載入時會停在亮燈靜態畫面並清除自動交棒 timer；只有按燈繩重播才開始完整光流，避免調整滑桿造成多個測試 timer 疊加、讓作品光提前完成。
- 首版 path stroke 把 `stroke-dasharray="0.24 0.76"` 設在 group；實際瀏覽器將它解析為極短實際單位並沿路徑重複，配合圓頭核心線後形成兩段可見的霓虹膠囊。此設定已移除。
- 修正版將每條 path 正規化為 `pathLength=1000`，使用單一 `260 2000` dash（pattern 長度大於整條路徑，不會重複），offset 由 `260 → -1000`。三層光改為低透明暖色並分別模糊 `30／18／10px`，核心 opacity 僅 0.22，只呈現牆面洗光而不顯示實體發光棒。
- 依 2026-07-13 回饋再增加作品光暈擴散：外／中／核心 blur 改為 `42／28／18px`，核心 opacity 降至 `0.19`。路徑、dash 長度與速度不變，只降低可辨識邊界。

## Pickmin 封面清晰度

- `pickmin-hero-1080.mp4` 實際為 `1080×750`、30fps，但平均視訊 bitrate 約 `302 kbps`；原始檔本身已是低碼率壓縮，CSS 無法還原被編碼丟失的細節。
- 先移除乳白材質層作用在影片上的 `backdrop-filter: blur(0.7px)`，中央霧膜透明度也降低；玻璃厚度改由邊緣高光與內陰影保留，不再模糊封面內容。
- 影片／poster 只加入非常輕微的 `contrast(1.045) saturate(1.025)`，用於抵銷霧膜造成的低對比，不使用可能產生白邊或閃爍的銳化濾鏡。
- 若仍覺得模糊，下一步必須替換成較高 bitrate 的原始輸出；將目前 302 kbps 檔案重新轉成高 bitrate 不會補回細節。
